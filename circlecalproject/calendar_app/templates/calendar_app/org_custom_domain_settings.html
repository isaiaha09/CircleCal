Standard Subdomain Setup

CircleCal uses your standard hosted subdomain by default (for example: https://yourbusiness.circlecal.app). This is the recommended setup and does not require DNS, Cloudflare, or website ownership.

What to do
1) Share your CircleCal hosted URL directly with clients.
2) Optionally paste your iframe embed code into your website (if you have one).
3) No DNS setup is required for standard subdomain usage.

Optional Branded Subdomain

This step is optional. Use it only if you want a branded booking URL like booking.yoursite.com. If you skip this, your hosted CircleCal URL above still works for sharing and embeds.

Branded subdomain setup has two DNS steps at your domain provider: (1) Cloudflare DCV verification record(s) (shown below), and (2) a CNAME so booking.yoursite.com points to CircleCal.

No website is required. You only need to own a domain if you choose this branded subdomain option.

Embed vs Subdomain
Embed = optional iframe code you can paste into your website (if you have one).
Booking subdomain = optional branded URL that points to CircleCal.
No website? Share your hosted CircleCal URL above — clients can still book normally.
Good to know: Changing your business name or your public URL (slug) does not require updating DNS or re-verifying your booking subdomain. You only need DNS changes if you change the actual subdomain (example: booking.yoursite.com → schedule.yoursite.com).
If you change your slug, just re-copy the embed code from the Services page (so your website uses the latest link).
